1. ANALYTICS PAGE (AnalyticsPage.tsx)
============================================================

DYNAMIC DATA SOURCES:
✅ Mean Behavior Score - from analyticsData.avg_safety_score
✅ Safety Gear Compliance - calculated from violations/workers ratio
✅ Alert Density Ratio - calculated from violations per worker per day
✅ Compliance Trend Chart - from dashboardAPI.getViolationsTimeline(180)
✅ Peak Violation Hours - from violationsAPI.list({limit: 500})
✅ Department Leaderboard - from workersAPI.list({limit: 1000})

API CALLS MADE:
1. dashboardAPI.getStats() - Overall dashboard statistics
2. dashboardAPI.getViolationsTimeline(180) - 6-month trend data
3. violationsAPI.list({limit: 500}) - For hourly analysis
4. workersAPI.list({limit: 1000}) - For department ranking

CALCULATED METRICS:

A) Mean Behavior Score:
   - Source: analyticsData.avg_safety_score
   - Backend: SELECT AVG(safety_score) FROM workers
   - Display: Dynamic value from database

B) Safety Gear Compliance:
   - Formula: 100 - (violations_per_worker * 10)
   - Calculation: total_violations / total_workers
   - Example: 100 violations / 15 workers = 6.67
             100 - (6.67 * 10) = 33.3% compliance
   - Display: Dynamic percentage

C) Alert Density Ratio:
   - Formula: (total_violations / (total_workers * 30)) * 100
   - Represents: Violations per worker per day as percentage
   - Example: 100 / (15 * 30) * 100 = 22.2%
   - Display: Dynamic percentage

CHARTS:

A) Compliance Trend Graph (Area Chart):
   - X-Axis: Months (from timeline data)
   - Y-Axis Left: Compliance % (85-100 range)
   - Y-Axis Right: Violations count
   - Data: 180 days of violation history
   - Dynamic: ✅ Fully dynamic from backend

B) Peak Violation Hours (Bar Chart):
   - X-Axis: Hours (00:00 - 23:00)
   - Y-Axis: Violation count
   - Data: Extracted from violation detected_at timestamps
   - Aggregation: Groups violations by hour
   - Dynamic: ✅ Calculated from real violation data

C) Department Safety Leaderboard:
   - Shows: All departments ranked by avg safety score
   - Data: Aggregated from worker safety scores
   - Display: Department name, worker count, score bar
   - Sorting: Highest score first
   - Dynamic: ✅ Real-time from worker database

3. BACKEND ENDPOINTS USED
============================================================

A) GET /api/analytics/dashboard
   Returns:
   - total_workers: Active worker count
   - total_violations: All violations
   - pending_violations: Pending status
   - approved_violations: Approved status
   - total_fines: Fine count
   - total_fine_amount: SUM of all fines (Rs.)
   - total_alerts: All alerts
   - unread_alerts: Unread count
   - active_emergencies: Emergency alerts
   - average_safety_score: AVG safety score
   - today_violations: Today's count

B) GET /api/analytics/violations-by-type
   Returns: Array of {type, count}
   Example: [{type: "no_hardhat", count: 15}, ...]

C) GET /api/analytics/violations-timeline?days=30
   Returns: Array of {date, count}
   Example: [{date: "2026-06-01", count: 5}, ...]

D) GET /api/analytics/fine-summary
   Returns:
   - total_amount: SUM of all fines
   - paid_amount: SUM of paid fines
   - pending_amount: SUM of unpaid fines
   - by_type: Array of {type, count, amount}

E) GET /api/analytics/safety-scores
   Returns: Top 20 workers by safety score
   Example: [{worker_name, department, safety_score}, ...]
