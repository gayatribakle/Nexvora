============================================================
PROBLEM IDENTIFIED
============================================================

The violations API endpoint (GET /api/violations) was NOT 
returning fine amount data. It only returned basic violation
information without linking to the Fine table.

Frontend Code (ViolationsPage.tsx line 64):
  fineAmount: v.fine_amount || 0,  // Always 0!

Backend API (violations.py):
  Was returning violation data WITHOUT fine information

============================================================
ROOT CAUSE
============================================================

Database Structure:
- Violation table: Stores violation details (type, status, etc.)
- Fine table: Stores fine amounts (linked via violation_id)
- Relationship: One violation can have one fine

The API was querying Violation table but not fetching the
related Fine data through the relationship.

============================================================

============================================================

File: backend/app/api/violations.py

============================================================
BUSINESS LOGIC
============================================================

When is fine_amount = 0?
1. Violation status is "pending" (not yet approved)
2. Violation status is "rejected" (no fine applied)
3. Violation was approved but fine creation failed (rare)

When is fine_amount > 0?
1. Violation status is "approved"
2. FineService.create_fine() was called successfully
3. Fine amount calculated from PenaltyRule.base_amount

Fine Creation Flow:
1. Violation detected by AI
2. Violation saved with status="pending"
3. Admin/Safety Officer approves violation
4. POST /api/violations/{id}/approve called
5. FineService.create_fine() creates fine
6. Fine amount calculated from penalty rules
7. Worker total_fine_amount updated
8. fine_amount now > 0 in API response
